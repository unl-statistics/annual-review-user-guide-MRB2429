Introduction

Elo ratings are a popular way for ranking competitors in head-to-head competitions. Each competitor has a certain rating that updates after each and every match depending on the outcome. Rating differences can be used to predict future match results too. This guide explains how to implement an Elo rating system in R using the PlayerRatings package. It also includes instructions for loading and preparing match data, calculating Elo ratings, visualizing ratings and interpreting the results. The workflow is based on the concepts described by Glickman and Jones.(Glickman and Jones 2024)

Prerequisites

Before beginning this guide, users should have:

  • R and RStudio installed
  • Basic familiarity with R (loading data and using packages)
  • A dataset containing player1, player2, and result columns
  • Match data arranged in chronological order

Ensuring these requirements are met will allow the Elo rating system to function correctly.

Step 1: Data Preparation

Before model implementation, the dataset must be cleaned and formatted properly. Player names should be consistent across all observations, and matches must be ordered chronologically to ensure accurate rating updates.

library(dplyr)

# Placeholder dataset for demonstration
elo_data <- data.frame(
  player1 = c("Player A", "Player B", "Player A", "Player C"),
  player2 = c("Player B", "Player C", "Player C", "Player D"),
  result  = c("player1", "player2", "player1", "player2")
)

# Encode results numerically for Elo
elo_data <- elo_data %>%
  mutate(score1 = ifelse(result == "player1", 1, 0),
         score2 = ifelse(result == "player2", 1, 0)) %>%
  select(player1, player2, score1, score2)

elo_data

2 Player B Player C      0      1
3 Player A Player C      1      0

Note

The dataset above is a placeholder created directly in R for demonstration purposes. In practice, users should replace it with their own match data containing the columns player1, player2, and result.

Step 2: Loading Required Package

The Elo rating system is implemented using the PlayerRatings package (Stephenson and Sonas 2020). The PlayerRatings package provides functions to help dynamically update player ratings

library(PlayerRatings)

Step 3: Parameter Specification

The K-factor controls how much a player’s rating changes after each match. The higher the K-factor means the faster the ratings change. A lower K-factor makes ratings a lot more stable.

The choice of the K-factor number depends a lot on the amount of data available. If there is aren’t a lot of matches or the player is new then having a higher K-factor, like 32, is useful because it allows ratings to adjust quickly. Then on the other hand, if there is a more established player with a lot more matches, a lower K-factor, like 16-24, is better because ratings don’t change as quickly.

k_factor <- 32

Step 4: Model Estimation or Updating

The Elo model updates ratings after each match by comparing what actually happened to what was expected. If a player performs better then expected it goes up and if not it goes down.

This process happens one match at a time, so ratings change as more data is added. Because of this, the matches must be in chronological order to get correct results.

Step 5: Evaluation

After calculating the ratings, it is useful to evaluate and visualize the results. This helps identify which players are ranked higher and whether the ratings appear reasonable.

The figure below shows an example of Elo ratings for a set of players.

players <- c("Player A", "Player B", "Player C", "Player D")
ratings <- c(1600, 1500, 1450, 1300)

barplot(ratings,
        names.arg = players,
        xlab = "Players",
        ylab = "Elo Rating",
        main = "Example Elo Ratings")

As shown in Figure 1, players with higher ratings are considered stronger. These visualizations can help check that the model results make sense and are easy to interpret.

Step 6: Interpretation

After calculating the Elo ratings, you can use them to tank players and also estimate changes of winning future matches.

  • Players with the same rating have about a 50% chance of winning.
  • A player with 200 points higher than their opponent would win about 75% of the time.
  • A player with 400 points higher than their opponent would win about 90% of the time.

Larger differences in ratings indicate a higher likelihood that the stronger player will win. You should also check that rating changes are reasonable and stable over time, especially for top players.

Common Questions

Why are ratings not changing much?
This may be due to a low K-factor.

Why do results look incorrect?
Ensure matches are in chronological order.

Can I include ties?
Yes, but they must be encoded correctly depending on the package.
